STRUCT 1 — ngx_http_request_s        VERDICT: BROKEN at/after uri_changes
================================================================================
Call sites (all routed through the C shim, NOT bindgen accessors):
  src/metric_source/span_start.rs  r_internal           (production)
  src/traces/ctx.rs                r_internal, r_filter_finalize (production)
  src/lib.rs:1133                  r_header_only         (test/test-support
                                   otel_status content handler; shimmed by H3F10)

clang record layout (struct-absolute byte:bit-range):
           0 | struct ngx_http_request_s
        1200 |   in_port_t port
   1202:0-15 |   unsigned int count
    1208:0-3 |   unsigned int uri_changes
    1212:0-0 |   unsigned int header_only
    1212:6-6 |   unsigned int internal
    1213:0-0 |   unsigned int filter_finalize
             | [sizeof=1408, align=8]

Trigger PRESENT: `in_port_t port` (byte 1200, a non-bitfield unsigned short)
shares the leading 4-byte allocation unit with the first bitfield `count`
(1202:0-15). The C ABI pads in struct-absolute coordinates; bindgen re-packs
into a fresh unit-local `_bitfield_1` and ignores the 16 bits `port` consumes,
so every accessor at/after uri_changes lands 2 bits low.
  field            real unit-bit (clang)   bindgen unit-bit   delta
  header_only      80   (1212:0)           78                 +2  BROKEN
  internal         86   (1212:6)           84                 +2  BROKEN
  filter_finalize  88   (1213:0)           86                 +2  BROKEN
  (unit base = `count` at byte 1202; e.g. header_only 1212:0 -> (1212-1202)*8 = 80)
=> MUST use the shim. Done (r_internal / r_filter_finalize in H3F1;
   r_header_only in H3F10).

SUMMARY
  Bindgen bitfield-accessor sites in src/ (production + test): 4 nginx structs.
    ngx_http_request_s    BROKEN  -> shimmed (r_internal / r_filter_finalize /
                                    r_header_only).
    ngx_event_s           SAFE    -> bindgen accessors (incl. exporter/mod.rs:515).
    ngx_variable_value_t  SAFE    -> bindgen SETTERS (lib.rs trace_id /
                                    parent_sampled variable handlers).
    ngx_buf_s             SAFE    -> bindgen setters (test-gated).
  Only ngx_http_request_s carries the allocation-unit-sharing trigger (a
  non-bitfield member co-resident in the bitfield's leading allocation unit);
  the other three open their bitfield region on a clean aligned boundary and
  match the C ABI bit-for-bit, getters and setters alike.
