SOUNDNESS FIX (exposed by F1 — not in the original findings, but real)
================================================================
The custom BIO_METHOD was a single process-global shared across ALL inner-IO
types `I`. Its read/write callbacks are `I`-monomorphized (bio_read::<I> casts
the BIO `data` slot to `*mut BioCtx<I>`); sharing one method across different
`I`s type-puns that pointer. Production has a single `I` so it never bit, but
the new multi-`I` test binary (TestIo + SpinIo) deterministically corrupted
later handshakes. Replaced with a TypeId-keyed registry (one method per concrete
`I`; requires `I: 'static`). A `static` inside the generic fn does NOT fix it
(Rust shares such statics across monomorphizations). A process-wide
S_SERVER_TEST_LOCK additionally serializes the s_server-backed test harnesses.

================================================================
A0.5 FLAVOR-GUARD — FIX + STOP-AND-ASK
================================================================
FIX (committed): detection changed from `contents.contains("NGX_DEBUG")` to a
   whitespace-tolerant positive `#define NGX_DEBUG 1` parse. Verified on macOS:
   a real debug tree (objs-debug, which has `#define NGX_DEBUG 1`) under the
   release profile STILL triggers the hard error; an ifndef-only tree does NOT
   (mutation 4, unit-level, both halves executed).

STOP-AND-ASK (premise divergence — the brief's diagnosis was incomplete):
   The brief hypothesised the false positive was `contains` matching
   `#ifndef NGX_DEBUG`. On debian-vm that is NOT the cause: objs-debug genuinely
   has `#define NGX_DEBUG 1`. The real cause of `make test` aborting is that the
   integration scripts (tests/integration/run.sh etc.) run `cargo build --release`
   against the NGINX_BUILD_DIR that `make test` hands them via TEST_ENV =
   objs-debug (the DEBUG tree). That is a genuine release-profile + debug-tree
   pairing, which the guard correctly flags. The `ngx-debug` cargo profile used
   by the nginx-module build inherits "dev" and reports PROFILE=debug (fine); it
   is specifically the integration scripts' `--release` build that trips it.

   The positive-define fix is correct and prescribed, but it does NOT by itself
   unblock `make test`, because the integration `--release` build against the
   debug tree is a legitimate guard trigger. Unblocking requires either:
     (a) the integration scripts to build against a release tree (objs-release)
         — FORBIDDEN by the brief ("zero changes to existing integration scripts");
     (b) make test's TEST_ENV to point the scripts at objs-release (Makefile
         change — within scope, but changes test wiring and may mismatch the
         debug nginx binary the scripts also use); or
     (c) relaxing the guard for the release-against-debug-tree case to a WARN
         rather than a hard abort (weakens the protection A0.5 deliberately added).
   This is a design decision; per the unconditional STOP-AND-ASK trigger on a
   false premise, it is surfaced rather than decided unilaterally. The escape
   hatch (NGX_OTEL_ALLOW_FLAVOR_MISMATCH=1) makes the suite green today.

================================================================
DESIGN NOTE — how TLS enters the gRPC connection (F3)
================================================================
GrpcTransport::send/send_logs/send_traces each do, on lazy connect:
    let io = self.connect_io().await?;          // was: self.connector.connect(...)
    hyper::client::conn::http2::handshake::<_, _, tonic::body::Body>(NgxExecutor, io)
connect_io opens the raw TCP stream via the connector, then — for an https://
endpoint — calls the shared wrap_tls_io(raw, ctx, insecure, host, Some(b"h2")),
which builds a TlsNgxConnIo (one engine, both transports), installs IP-SAN
verification for IP literals, and sets ALPN "h2" via SSL_set_alpn_protos so the
collector negotiates HTTP/2 over TLS (RFC 7540 §3.3). The boxed Box<dyn TlsOrPlain>
satisfies the h2 handshake's IO bounds via hyper's blanket Read/Write impl. No
tonic connector seam was needed — the seam is the IO handed to the h2 handshake.
